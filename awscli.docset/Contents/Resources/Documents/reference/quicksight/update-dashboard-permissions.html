update-dashboard-permissions¶

Description¶

Updates read and write permissions on a dashboard.

See also: AWS API Documentation

See 'aws help' for descriptions of global parameters.

Synopsis¶ 

  update-dashboard-permissions
--aws-account-id <value>
--dashboard-id <value>
[--grant-permissions <value>]
[--revoke-permissions <value>]
[--cli-input-json <value>]
[--generate-cli-skeleton <value>]

Options¶

--aws-account-id (string)

The ID of the AWS account that contains the dashboard whose permissions you're updating.

--dashboard-id (string)

The ID for the dashboard.

--grant-permissions (list)

The permissions that you want to grant on this resource.

Shorthand Syntax:

Principal=string,Actions=string,string ...

JSON Syntax:

[
  {
    "Principal": "string",
    "Actions": ["string", ...]
  }
  ...
]

--revoke-permissions (list)

The permissions that you want to revoke from this resource.

Shorthand Syntax:

Principal=string,Actions=string,string ...

JSON Syntax:

[
  {
    "Principal": "string",
    "Actions": ["string", ...]
  }
  ...
]

--cli-input-json (string) Performs service operation based on the JSON string provided. The JSON string follows the format provided by --generate-cli-skeleton. If other arguments are provided on the command line, the CLI values will override the JSON-provided values. Output¶

DashboardArn -> (string)

The Amazon Resource Name (ARN) of the dashboard.

DashboardId -> (string)

The ID for the dashboard.

Permissions -> (list)

Information about the permissions on the dashboard.

(structure)

Permission for the resource.

Principal -> (string)

The Amazon Resource Name (ARN) of an Amazon QuickSight user or group, or an IAM ARN. If you are using cross-account resource sharing, this is the IAM ARN of an account root. Otherwise, it is the ARN of a QuickSight user or group. .

Actions -> (list)

The action to grant or revoke permissions on, for example "quicksight:DescribeDashboard" .

(string)
